The Problem

The Paris Review website lacks utility. Their is no way to keep track of stories you've read or exclude them from new searches.

The Solution

An app that searches the archives of theparisreview.org while excluding the titles in your library from results.

The Plan

I first focused on the data structures, deciding on a NoSQL database of three collections. A collection of stories for each user. A global collection of stories that users query by various parameters. And a global collection of story summaries. Entries in each collection sharing the same title received a shared 4 digit identifier.

Next up was acquiring the database of titles from theparisreview.org. The structure of the website made scraping the urls quite simple. All fiction stories are listed under a directory named fiction and assigned a 4 digit url parameter corresponding to the sequence of published stories with 0001 being the first story published in The Paris Review.

Once I had the array of URLs I mapped a new array of objects containing the key value pairs that would suffice for my uses' query requirements. This array served as the collection of searchable stories.
